Contexto
02 / 14
Problema y solución

Digitalización del ecosistema operativo de una liga deportiva

Problema

  • Gestión manual de ligas, equipos y jugadores.
  • Poca trazabilidad en altas y bajas.
  • Partidos, resultados y clasificación dispersos.
  • Usuarios sin acceso rápido a su información.

Solución

  • Panel administrativo con roles.
  • Workflow STAFF → ADMIN para validar jugadores.
  • Portal público dinámico.
  • Favoritos y calendario para usuario.

Base de datos
04 / 14
Modelo relacional

La base de datos estructura toda la competición

El sistema utiliza una base de datos relacional en MySQL para conectar usuarios, ligas, equipos, jugadores, entrenadores, partidos y clasificaciones.

Tablas puente clave

  • equipo_liga → relaciona equipos y ligas.
  • equipo_jugador → plantilla oficial.
  • entrenador_equipo → asignación staff.
  • clasificacion → puntos, goles y posiciones.

Decisiones técnicas

  • Relaciones N:M mediante tablas intermedias.
  • Estados para trazabilidad de jugadores.
  • Separación entre competición y usuarios.
  • Modelo preparado para escalar temporadas.

13 / 14
Conclusiones

Una plataforma sólida para la gestión deportiva digital

Gwinnett Soccer League Hub consolida en una única plataforma la gestión de ligas, equipos, jugadores, partidos y usuarios mediante una arquitectura modular, organizada y preparada para escalar.

Centralización

Unificación de procesos administrativos, competitivos y operativos en una sola plataforma accesible.

Arquitectura

Separación clara entre frontend, routing, controladores, modelos y base de datos para facilitar mantenimiento.

Escalabilidad

Diseño modular preparado para incorporar nuevos roles, funcionalidades y competiciones sin rehacer el sistema.
